# Contributing to Simple Shopping List

First off, thank you for considering contributing to this project! It’s people like you who make the open-source community such an amazing place to learn, inspire, and create.

As an educational repository, we welcome all types of contributions—from fixing typos in the documentation to adding cool new features!

---

## 🚀 How Can I Contribute?

### 1. Reporting Bugs & Suggesting Features
*   Check the [Issues](https://github.com/Sahaab-Ji/Simple-Shoping-List/issues) tab to see if your bug or feature idea has already been reported.
*   If not, open a new issue. Clearly describe the problem or the feature you want to see added, and include steps to reproduce it if it's a bug.

### 2. Improving the Code or Documentation
Want to add a feature (like `localStorage` persistence, a dark mode toggle, or an edit button)? Follow these steps:

1. **Fork** this repository to your own account.
2. **Clone** your fork locally:
   ```bash
   git clone [https://github.com/Sahaab-Ji/Simple-Shoping-List.git](https://github.com/Sahaab-Ji/Simple-Shoping-List.git)

### 3. Create a new branch for your feature or bug fix:

Bash
git checkout -b feature/your-feature-name

### 4. Make your changes and commit them with a clear message:

Bash
git commit -m "feat: add local storage to save items"

### 5. Push the changes to your GitHub fork:

Bash
git push origin feature/your-feature-name

### 6. Open a Pull Request (PR) against our main branch.

📜 Ground Rules & Code Style
Keep the HTML, CSS, and JavaScript clean and well-commented so beginners can easily read and understand it.

Be respectful and supportive of other contributors in issues and pull request discussions.

Thank you for helping make this project better! Don't forget to ⭐ the repo if you enjoy working on it!
